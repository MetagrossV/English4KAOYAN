**⚠️ 警告详情：`startTimer()`（第151-162行）**

```javascript
startTimer() {
  const start = Date.now() - this.data.elapsedSeconds * 1000;
  this.setData({ timerStart: start });
  const interval = setInterval(() => {
    // ...
  }, 1000);
  this.setData({ timerInterval: interval }); // 第161行
}
```

- **问题**：在 `setInterval` 外部先设置 `timerStart`，回调内部再设置 `elapsedSeconds` 和 `timerDisplay`。同时 `timerInterval` 在 `setInterval` 启动后才保存。
- **影响**：极低。小程序单线程执行，不会导致竞态条件。但第161行额外的 `setData` 会触发一次不必要的页面渲染。
- **修复建议**：将 `timerInterval` 的赋值合并到初始 `setData` 中：
  ```javascript
  const interval = setInterval(...);
  this.setData({ timerStart: start, timerInterval: interval });
  ```

**⚠️ 警告详情1：`wx:key="index"`（第44行）**

```xml
<view wx:for="{{sentences}}" wx:key="index" ...>
```

- **问题**：使用数组索引作为 `wx:key`。小程序官方文档指出，如果数组数据顺序发生变化，使用 `index` 作为 key 会导致渲染状态错乱。
- **实际影响**：句子顺序是固定的，不会动态变化，因此当前风险较低。
- **修复建议**：如果追求最佳实践，可使用句子内容本身作为 key（需确保句子唯一），或保持现状。

**⚠️ 警告详情2：`{{sentences[selectedSentenceIndex]}}`（第98行）**

```xml
<view class="original-sentence">
  <text>{{sentences[selectedSentenceIndex]}}</text>
</view>
```

- **上下文**：该节点位于 `wx:if="{{showTranslatePanel}}"` 内部。
- **问题**：`selectedSentenceIndex` 初始值为 `-1`。虽然理论上 `showTranslatePanel` 为 `false` 时不会渲染，但如果条件渲染与状态更新存在时序差，可能短暂访问 `sentences[-1]`（返回 `undefined`）。
- **实际影响**：微信小程序中 `undefined` 在 WXML 中渲染为空字符串，不会崩溃，风险极低。
- **修复建议**：可增加兜底表达式：`{{sentences[selectedSentenceIndex] || ''}}`
