## 二、阅读真题获取方案（二选一）

### 方案 A：从现有 PDF 解析阅读部分（推荐）
- 已有 `pastpapers.cn` 的32套PDF链接，每个PDF包含完整试卷（含阅读）
- 使用PDF解析工具提取 "Reading Comprehension" / "Text 1-4" 的原文文本和题目
- 优点：来源稳定，覆盖2010-2025年完整
- 缺点：PDF解析可能有格式问题（换行、分页），需要人工清洗

### 方案 B：寻找已整理的文本数据源
- 搜索发现：知乎/B站/考研论坛有用户整理的阅读真题文本（如逐句翻译版）
- 夸克网盘有2010-2024年Word版真题，可复制文本
- 优点：文本更干净，解析工作量小
- 缺点：来源分散，需要逐个确认，版权风险需规避

> **建议**：先做方案 A（解析PDF），如提取质量不佳，再用方案 B 补充。  
> 每套试卷包含 **4篇阅读 × 5题 = 20题**，共32套 = 640道阅读题。

## 三、做题交互方式设计（核心确认）

你提到的思路非常好，我将其细化为以下功能模块：

### 3.1 核心做题流程

```
┌─────────────────────────────────────────────────────┐
│  阅读页面                                             │
│  ───────────────────────────────────────────────────  │
│  【文章区域】                                          │
│  ┌──────────────────────────────────────────────┐   │
│  │  ① 点击文中某句 → 弹出五色标记面板               │   │
│  │     ┌──┬──┬──┬──┬──┐                           │   │
│  │     │ Q1 │ Q2 │ Q3 │ Q4 │ Q5 │  ← 5种颜色对应5题   │   │
│  │     └──┴──┴──┴──┴──┘                           │   │
│  │  ② 已标记的句子显示背景色（如Q1=浅红，Q2=浅黄）   │   │
│  │  ③ 长按句子 → 输入/编辑翻译（显示在英文下方）    │   │
│  │                                                 │   │
│  │  The researchers first established...             │   │
│  │  [研究人员首先确定了...]  ← 用户翻译              │   │
│  └──────────────────────────────────────────────┘   │
│  【题目区域】                                          │
│  ┌──────────────────────────────────────────────┐   │
│  │  21. What did the researchers first do?       │   │
│  │  ○ A. ...                                    │   │
│  │  ○ B. ...                                    │   │
│  │  ○ C. ...                                    │   │
│  │  ○ D. ...                                    │   │
│  └──────────────────────────────────────────────┘   │
│  【底部操作栏】                                         │
│  [提交答案]  [查看标记]  [AI解析]  [收藏句子]          │
└─────────────────────────────────────────────────────┘
```

### 3.2 五色标记系统

| 颜色 | 对应题目 | 用途 |
|------|---------|------|
| 浅红 | Q1 | 标记第1题的关键句/线索句 |
| 浅黄 | Q2 | 标记第2题的关键句/线索句 |
| 浅绿 | Q3 | 标记第3题的关键句/线索句 |
| 浅蓝 | Q4 | 标记第4题的关键句/线索句 |
| 浅紫 | Q5 | 标记第5题的关键句/线索句 |
| 无标记 | — | 未被标记的普通句子 |

- 支持：点击句子 → 选择颜色 → 标记/取消标记
- 支持：同一句子可标记多个颜色（如某句同时是Q1和Q3的线索）
- 标记后，文章顶部显示迷你图例（已标记的颜色分布）

### 3.3 逐句翻译输入

- 点击句子右侧的 "译" 按钮，或长按句子
- 展开输入框，用户输入自己的中文翻译
- 提交后，翻译显示在英文下方（小字号，灰色）
- 支持修改/删除
- 未来AI复盘时，会对比用户翻译和正确翻译，判断是否是**翻译理解错误**导致做错

### 3.4 错题复盘与 AI 解题思路

用户提交答案后，系统立即判断对错。做错时弹出复盘面板：

```
┌────────────────────────────────────────┐
│  ❌ 第3题 做错                          │
│  ──────────────────────────────────────  │
│  你的答案：B                             │
│  正确答案：C                             │
│                                         │
│  📍 你标记的关键句（Q3-浅绿）：           │
│     "The experiment showed that..."      │
│                                         │
│  📝 你的翻译：                           │
│     "实验表明大象可能会用嗅觉..."         │
│                                         │
│  🤖 AI 错误分析：                        │
│     你的翻译准确，但题目问的是"secondly"  │
│     你标记的句子只回答了前半部分，        │
│     遗漏了后半句 "assess the quality"    │
│     导致误选B。建议关注题干中的并列结构。  │
└────────────────────────────────────────┘
```

AI 分析维度：
1. **翻译错误**：用户翻译与原文偏差较大，导致理解错误
2. **定位错误**：标记的关键句不是正确线索句（找错地方）
3. **理解偏差**：翻译正确，但逻辑推理错误（如过度推断、偷换概念）
4. **未标记**：完全没有标记关键句，凭感觉选

## 五、技术方案

### 5.1 技术栈建议
- **前端**：微信小程序原生框架（WXML + WXSS + JS）
- **后端**：云开发（云函数 + 云数据库），无需自己搭建服务器
- **数据存储**：JSON格式存储文章、题目、用户标记和翻译
- **AI能力**：调用大模型API（如Kimi/通义千问）生成解题思路

## 七、需要你确认的关键问题

1. **五色标记交互**：点击句子选颜色是否OK？还是想要更直观的"先选颜色再涂抹句子"方式？

2. **翻译输入方式**：是逐句翻译（每句独立输入），还是段落翻译？逐句更适合错题定位，但输入量较大。

3. **阅读真题范围**：是否只覆盖2010-2025年？还是也包含更早的年份？

4. **AI解析触发方式**：是做错题立即自动分析，还是用户手动点击"AI解析"才生成？（自动可能消耗较多token）

5. **小程序名称**：有想好的名字吗？如没有，建议几个备选（如"考研阅读助手"、"研英语"等）。
